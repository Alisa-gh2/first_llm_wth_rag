# first_llm_wth_rag

Это мой первый масштабный проект в **ML**, где реализован бэкенд на **FastAPI** для работы на базе LLM с **RAG-пайплайном** для ответов на вопросы по учебным документам. 

# Корпус документов и постановка задачи

В качестве основной предметной области будут **учебные тексты**, то есть файлы лекций, конспектов и прочего с целью суммаризации, подготовки блиц-опросов и других методов, позволяющих оптимизировать и усовершенствовать процесс изучения и усвоения материалов.
# Архитектура RAG-пайплайна

**Загрузка документов** происходит через  `PyPDF2`. 

**Разбиение на чанки**: текст мы делим на фрагменты по 500 символов. Перекрытие 50 символов.

**Эмбеддинги**: далее каждый чанк преобразуется в вектор при помощи модели all-MiniLM-L6-v2, взятой из Sentence Transformers.

**Векторное хранилище**: векторы хранятся в FAISS-индексе для обеспечения быстрого поиска. 

**Поиск**: для поиска пользователь формирует запрос, после чего идет поиск 5 наиболее похожих (подходящих) чанков (L2-расстояние).

**Генерация ответа**: чанки, которые были найдены как 5 наиболее похожих относительно запроса,  вставляются в промпт и отправляются в LLM-модель через OperRouter API. На основе контекста модель строит ответ, после чего он отправляется обратно пользователю. 

Если по какой-то причине модель не сможет построить ответ, она выдаст соответствующую ошибку, например, что недостаточно информации. 

### Выборка текстов находится в папке [/docs](https://github.com/Alisa-gh2/first_llm_wth_rag/tree/main/docs). 

### В папке [/colab](https://github.com/Alisa-gh2/first_llm_wth_rag/tree/main/colab) находится файл .ipynb, на котором в Google Colab сделан RAG-пайплайн. 

# Установка

## Для установки на Linux:

```
git clone https://github.com/Alisa-gh2/first_llm_wth_rag.git
cd F1r5t_LLM_W1th_R4G
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```
Скопируйте .env.example в .env, а затем укажите в нем свой API_KEY. 

Загрузите свои докумнеты для модели (txt, html, md, pdf и др.) в папку docs/.

Затем запустите сервер:
```
uvicorn main:app --reload
```
После запуска сервис будет доступен по ссылке http://127.0.0.1:8000.

## Для установки на Windows:

Для работы необходим Python. Примечательно, что если в настройках Windows подключен прокси, возможно зависимости не смогут корректно настроиться. Рекомендуется отключить прокси на время настройки зависимостей. 

Клонируем репозиторий: 
```
git clone https://github.com/Alisa-gh2/first_llm_wth_rag.git
```

Переходим в папку проекта:
```
cd \first_llm_wth_rag
```

Далее требуется создать виртуальное окружение:
```
python -n venv venv
venv/Scripts/activate
```

Затем требуется настроить зависимости:
```
pip install -r requirements.txt
```

Установка может занять 5-7 минут, так как пакеты весят несколько ГБ. 

После установки зависимостей требуется из файла .env.example взять строку OPENROUTER_API_KEY=ваш_ключ и вставить в новый файл с названием .env, а также записать туда свой API-ключ с OperRouter.

Требуется также убедиться в том, что в папке /docs присутствуют файлы. 

Для запуска сервера используем команду:

```
uvicorn main:app --reload
```

После запуска сервера будет доступен SwaggerUI по ссылке http://127.0.0.1:8000/docs.

# Ограничения и возможные улучшения

Есть ограничение за длину запроса пользователя. Его пользовательский ввод должен быть длиной не более 500 символов. 

Одним из основных ограничений является фиксированный размер чанков в 500 символов. Соответственно, чем больше размер чанка - тем лучше, поскольку происходит минимизация разрывов смысловых блоков. 

Также в данном случае модель смотрит лишь файлы pdf, однако технически присутствует возможность улучшения с целью получения возможности извлекать данные из других форматов, таких как txt, md, odt, ods, rtf, docx и других.

Также заметим, что у нас есть ограничение на длину контекста. Сейчас у нас используется top 5 чанков, что теоретически может не покрыть информацию, которую мы ищем. 

В будущем можно увеличить объем корпуса текстов, улучшить работу с чанками, сделать к примеру не топ 5, а топ 10 для большего покрытия. 

Еще можно экспериментировать с изменением системного промпта, что должно положительно сказаться на качестве отклича модели за запросы.
